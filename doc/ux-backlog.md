# UX Backlog（可跟踪条目）

本文档把 `doc/ux-improvements.md` 中的 P0/P1 事项拆成可直接创建 Issue / 任务卡的条目。

字段说明：

- 优先级：P0/P1
- 目标：解决的核心体验问题
- 影响范围：涉及页面/模块
- 交付物：可交付的具体成果
- 验收：可验证的验收标准

## P0

### UX-P0-001：Workspace 展示阶段进度与失败原因

- 目标：用户能清楚知道任务处于哪个阶段、失败原因是什么
- 影响范围：Workspace、轮询、任务详情展示
- 交付物：
  - 状态区展示：阶段（PARSING/DOWNLOADING/TRANSCRIBING/…）、message（失败态）
  - 操作区：复制错误 / 重试 / 去设置（按情况显示）
- 验收：
  - FAILED 任务在页面上可见失败 message，不依赖 toast
  - 对失败任务，至少存在一个可执行动作（重试或去设置）

### UX-P0-002：首页未配置模型的显式引导

- 目标：新用户不被动 toast 打断，且能明确下一步配置路径
- 影响范围：首页、设置页入口
- 交付物：
  - 首页“未配置”提示卡片（说明+按钮跳转设置）
  - 可见的后端状态提示（已连接/初始化中）
- 验收：
  - 未配置时，创建按钮禁用并显示原因与入口
  - 配置完成后无需刷新即可继续创建

### UX-P0-003：空状态统一组件

- 目标：减少认知成本，提高空状态可操作性
- 影响范围：首页历史区、History 页、Workspace
- 交付物：
  - 统一空状态组件（无任务/无结果/无筛选结果）
  - 每种空状态包含明确的主按钮动作（去创建/清空筛选/刷新）
- 验收：
  - 三个页面空状态风格一致且都有动作按钮

## P1

### UX-P1-001：搜索统一（标题/URL/标签）

- 目标：历史检索一致且高效
- 影响范围：首页历史侧栏、History 页
- 交付物：
  - 统一搜索策略：标题/URL/标签
  - 展示命中数；无结果时给出建议（如清空筛选）
- 验收：
  - 同一关键字在两个入口结果一致

### UX-P1-002：历史批量操作（多选）

- 目标：大量任务管理更高效
- 影响范围：History 页
- 交付物：
  - 多选模式（checkbox 或批量选择）
  - 批量删除、批量打标签（导出可后续拆分）
- 验收：
  - 选中多条后可一次性完成删除或添加标签

### UX-P1-003：播放与笔记联动（时间戳跳转）

- 目标：提升“看视频做笔记”的沉浸体验
- 影响范围：Workspace 播放器、笔记渲染
- 交付物：
  - 识别时间戳并点击跳转播放器时间
- 验收：
  - 点击时间戳，播放器跳转到对应时间并继续播放/暂停状态保持一致
